Release history

Changelog

All notable changes to LexiPath Desktop, from newest to oldest. Full release assets on the GitHub Releases page.

  1. v1.0.7 March 29, 2026 Latest Bug fix
    • fix Videos now play inside the app again — the "Watch on YouTube" workaround from v1.0.6 is replaced with real in-app playback. Opening theater mode now streams the video directly inside LexiPath without any browser redirect.
    • feat New backend endpoint proxies the YouTube CDN stream through the local Django server, bypassing iframe embedding restrictions (Error 153) entirely — the browser talks to localhost, not youtube.com.
    • perf Extracted CDN URLs are cached for 30 minutes — seeking works instantly without re-calling yt-dlp on every Range request. Falls back to the YouTube link if the backend is unreachable.
    View on GitHub →
  2. v1.0.6 March 28, 2026 Feature Feature Bug fix
    • feat Hover actions on video cards — hovering a card now reveals two icon buttons: ✎ (edit) and ✕ (delete), hidden by default and fading in on hover so the card layout stays clean during normal browsing.
    • feat Quick edit form on card — clicking ✎ opens an inline overlay with pre-filled title and YouTube URL inputs. Escape cancels, Enter or Save commits. No need to open the theater modal to fix a typo or wrong URL.
    • feat Edit video URL — the URL field in the quick edit form is fully editable. Changing it re-derives the video ID, thumbnail, and resets the offline download state on the backend.
    • feat Delete confirm on hover — clicking ✕ shows the same confirmation dialog as inside the theater modal, directly on the card.
    • feat YouTube iframe embed replaced with thumbnail + external link — the theater modal now shows the video thumbnail with a "Watch on YouTube" button that opens the video in the system browser, avoiding cross-origin embed errors (Error 153).
    • fix Changing a video's YouTube URL now correctly updates the thumbnail shown on the card (previously only the title could be changed and the thumbnail stayed stale).
    View on GitHub →
  3. v1.0.5 March 28, 2026 Performance
    • perf Startup time reduced by 1–3 seconds on every launch after the first. The server now hashes all migration filenames and skips the Django migrate step entirely when nothing has changed — the hash is stored in .migration_version in the user data directory and updated only after a successful migrate.
    View on GitHub →
  4. v1.0.4 March 28, 2026 Bug fix
    • fix JWT access token expiry no longer breaks the session. The app now silently re-authenticates in the background when the 60-minute token window closes — learning paths and all API calls continue without interruption.
    View on GitHub →
  5. v1.0.3 March 28, 2026 Feature
    • feat Full UI internationalisation — LexiPath now ships in 6 languages: English, Brazilian Portuguese, Spanish, German, Italian, and French.
    • feat Language selector added to the dashboard header. Choice persists across sessions via localStorage.
    View on GitHub →
  6. v1.0.2 March 28, 2026 Bug fix
    • fix Updated yt-dlp to handle recent YouTube format changes — video downloads were failing for some content after a YouTube-side update.
    • perf Reduced installer sizes across all platforms through maximum compression and bundle refinement (LZMA/xz, UPX on Linux).
    View on GitHub →
  7. v1.0.1 March 27, 2026 Feature Bug fix
    • ci GitHub Actions CI — automated builds on Windows, macOS, and Linux triggered on every version tag.
    • feat Auto-updater via electron-updater. Windows and Linux download and apply updates silently in the background; macOS opens the releases page.
    • fix Videos stuck in "downloading" state no longer stay frozen after a restart — status resets to error on launch so you can retry.
    View on GitHub →
  8. v1.0.0 March 27, 2026 Initial release
    • feat First public release. Electron shell wrapping a local Django backend — the entire app runs on your machine with no server and no account required.
    • feat Offline YouTube video download and playback via yt-dlp. Downloaded files stored in the OS user data directory and survive app updates.
    • feat Automatic local authentication on first launch — no email, no password, no sign-up.
    • feat Native installers for Windows (.exe), macOS (.dmg, Apple Silicon), and Linux (.AppImage).
